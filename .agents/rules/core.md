# 项目通用规则

- 始终使用中文回复用户，但技术专有名词可保留英文（如 API、Python、DTO等）。
- 使用项目既有风格，不引入新风格，包括代码、文档或交互，默认延用项目中已有的格式、缩进、命名习惯。
- 尽可能少地输出内容，仅提供高信息密度回复，禁止无效寒暄、过度铺垫，只输出对当前任务有直接帮助的信息。
- 遵守[《中文文案排版指北》](https://www.bookstack.cn/read/chinese-copywriting-guidelines/README.md)标准排版格式：中文与英文之间加空格，中文与数字之间加空格；全角标点与其他字符之间不加空格。全中文语境下必须使用全角标点。
- 在安装 Skill、Subagent、Commands、MCP、Hook 时，根据所在项目上下文规范目标路径（例如在 TRAE 项目中安装 Skill 时应放在 `.agents/skills/` 目录下）。特别注意：如果因适配此路径规范而移动了下载工具默认的安装位置，必须同步在项目根目录的 `skills-lock.json` 文件中手工补齐该工具的注册信息（包含 `source`、`sourceType` 等字段），以防止依赖记录丢失。
- 文档命名规范：必须全部使用小写字母，文档名需与文章内容主题紧密契合，多个单词间使用 `-` 分隔。命名应精简，不超过 4 个词汇（单词），以 3 个词以内为宜。
- Markdown 语法与渲染兼容性检查：中文全角标点（特别是中文句号 `。`）可能阻断加粗标记（`**`）的解析导致渲染失败。必须在加粗语法的外部添加半角空格隔离（如 `文本 **加粗** 文本`），或者将标点符号包含在加粗语法内部（如 `**加粗句子。**`）来确保完全兼容。撰写文档后，应习惯性通过类似 `markdown-formatter` 技能修正此问题。
- 中文双引号规范：统一使用中文双引号“”而非英文双引号。在撰写或修改文档时，必须检查并替换所有英文双引号为中文双引号。
