# 公众号文章写作流程

4 步完成高质量公众号文章：搜索资料 → 撰写文章 → 生成标题 → 排版优化

## Step 1: 搜索资料

使用 WebSearch 工具搜索主题相关资料：

```text
搜索要求：
1. 并行搜索多个来源（官方文档、X/Twitter、Reddit、技术论坛）
2. 优先获取最新资料（当月/当季）
3. 可启动多个并行 Task 加速搜索
4. 深度总结搜索结果
```

## Step 2: 撰写文章

**必须先读取用户的 CLAUDE.md 或 rules 下的规则获取写作风格**

撰写要求：

- 1500-2000 字
- 标题简单：标题不要太长，控制在 10 个字以内，建议使用一二级标题，最多使用到三级标题
- 前言：200-300 字，介绍文章的写作出发点
- 在“前言”后标注：`特别说明：本文是笔者与AI“结对编程”式写作的产物。AI是我的协作者，而我作为主导者，对文章质量负责。`
- 关键词/搜索词：4 到 5 个，如 AI、运维、SRE、Skill 等领域专业词汇
- 标题默认不使用序号，如`一、二、三、四、五`等
- 正文描述使用陈述语句，而不要使用人称，如我、你、他等
- 结构：效果展示 → 问题描述 → 步骤教学 → 升华总结

## Step 3: 生成标题

生成 5 个标题，特点：

- **痛点明确**：直击读者痛处（"还在手动..."）
- **数字吸引**：具体数字更有说服力（"3 分钟"、"5 个技巧"）
- **结果导向**：承诺具体收益（"效率暴涨 10 倍"）
- **情绪调动**：惊、神技、秘籍等词汇
- **悬念设置**：引发好奇心

## Step 4: 排版优化

优化建议：

1. **段落结构**：每段 3-5 行，重要数据单独成段加粗
2. **配图位置**：标题下方封面图、步骤后效果图、结尾总结图
3. **代码块**：前后留白，语法高亮
4. **金句**：单独成段，增强记忆点

## Step 5: 插入图片

在段落或文字中核实的位置插图图片，图片可以从互联网中搜索下载，也可以使用 nano banana 生成，但务必保存到本地，并使用本地图片。

插入图片要求：

- 图片大小适中，不要太大，否则会影响阅读体验
- 图片使用英文命名，保存到文档所在到上级目录，如文档 `blogger.md`，则其图片目录为 `../img-blogger/`，在文中中使用图片到相对路径引用，如 `![深圳中学](../img-blogger/shenzhen-senior-school.png)`
- 文件格式推荐保存为 png、jpg，也可以使用 svg、webp 等

## Step 6: 概述

在微信公众号、掘金等平台，在发布时需要输入文章的概述，请根据文章内容生成一个概述，要求：

- 严格遵守字数 80 左右，不超过 100 为宜
- 基于文章内容，提炼核心观点
- 文章概述与正文使用分隔符分割，分隔符为 `---`，概述可放在开头

## 示例

用户请求：

```text
帮我写一篇关于 Claude Code 使用技巧的公众号文章
```

执行流程：

1. Claude Code 炙手可热，想要一篇文章了解 Claude Code 使用技巧
2. 搜索 Claude Code 最新功能和使用技巧
3. 读取 CLAUDE.md 或 rules 下的规则获得写作风格
4. 撰写 1500 字左右的教程文章
5. 生成 5 个爆款标题供选择
6. 提供排版和配图建议，并预插入图片
7. 生成文章概述
