# 题目与知识点

## 1. 软件开发环境的集成机制

软件开发环境（SDE）是指支持软件的工程化开发和维护而使用的一组软件，由**软件工具集**和**环境集成机制**构成。

数据存储-流程调度-人机交互：环境信息库、过程控制与消息服务器、环境用户界面

## 2. 软件过程

制作软件产品的一组活动，以及结果。

软件描述、软件开发、软件有效性验证和软件进化。

## 3. 软件开发模型

极限编程：以人为中心，对费用控制严格的公司

水晶系列 Crystal：以人为中心，平衡**产出效率**与**易于运作**

开放源代码：程序开发人员在地域上分布很广。

SCRUM：明确定义的可重复方法过程只限于在明确定义的可重复的环境中。

功用驱动开发方法 FDD：短时间迭代，首席程序员和类程序员

自适应软件开发：3个非线性且重叠，猜测、合作、学习。

## 4. RUP

把软件开发声明周期划分为多个循环（Cycle），每个循环生成产品的一个新的版本。

4个顺序的阶段，迭代过程，每一次迭代的重点是对特定的用例进行部分实现。

- 初始：为系统建立业务模型并确定项目的边界，定义最终产品视图和业务模型，并确定系统范围。
- 细化：设计及确定系统的体系结构，制定工作计划及资源要求
- 构建：构造产品并继续演进需求
- 移交：交付产品给用户使用

## 7. M2M，Machine to Machine

是指数据从一台终端传送到另一台终端，共5个重要部分：
1. 智能化机器：能使机器“开口说话”；
2. M2M硬件：能够进行信息的提取，从各种机器设备那里获取数据，并传送到通信网络
3. 通信网络：将信息传送到目的地；
4. 中间件：在通信网络和IT系统间起桥接作用；
5. 应用：对所获数据进行加工，为决策和控制提供依据。

## 10. 企业信息集成

- 企业内部：4个方面
  - 技术平台集成：系统底层的体系结构、软件、硬件以及异构网络的特殊需求
  - 数据集成：简单、低成本、易于实施，但需要对系统内部业务深入了解。为了完成应用集成和业务集成
  - 应用系统集成：不同系统之间，数据和方法的共享
  - 业务过程集成：无缝连接，实现流程的协调运作和流程信息的充分共享。
- 企业外部：2个方面
  - 门户集成
  - 系统集成

## 11. 商标权

知识产权具有**地域性**。不同类别商品（产品）可以使用相同或类似的商标，但对于**驰名商标**来说，不能在任何商品中使用相同或类似的表示。

## 12. PKI 系统

PKI 系统：公钥基础设施，是一种用于管理和分发数字证书的系统。

- 证书机构 CA： 负责生成和签署数字证书
- 注册机构 RA： 负责验证申请数字证书用户的身份。

## 13. 4+1 视图模型

从5个不同视角来描述软件架构，每个视图只关心系统的一个侧面。
- 逻辑视图
- 开发视图：UML 中的实现视图，软件模块的组织和管理。
- 进程视图：运行特性，主要关注**非功能需求**，性能和可用性。
- 物理视图：UML 部署视图，拓扑结构、系统安装、通信问题
- **场景视图**：重要系统活动的抽象，使4个视图有机联系起来

## 14. 软件架构评估方式
分为3种：基于调查文件的方式、基于度量的方式、基于场景的方式。
基于场景的方式，有3种：
- SAAM：场景分析法，卡内基梅隆大学，非功能质量属性的架构分析方法，其主要输入是问题描述、架构描述、和架构描述文档。
- ATAM：架构权衡法
- CBAM：成本效益分析法
