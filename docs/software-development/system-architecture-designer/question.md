## 4. RUP

把软件开发声明周期划分为多个循环（Cycle），每个循环生成产品的一个新的版本。

4个顺序的阶段，迭代过程，每一次迭代的重点是对特定的用例进行部分实现。

- 初始：为系统建立业务模型并确定项目的边界，定义最终产品视图和业务模型，并确定系统范围。
- 细化：设计及确定系统的体系结构，制定工作计划及资源要求
- 构建：构造产品并继续演进需求
- 移交：交付产品给用户使用

## 7. M2M，Machine to Machine

是指数据从一台终端传送到另一台终端，共5个重要部分：

1. 智能化机器：能使机器“开口说话”；
2. M2M硬件：能够进行信息的提取，从各种机器设备那里获取数据，并传送到通信网络
3. 通信网络：将信息传送到目的地；
4. 中间件：在通信网络和IT系统间起桥接作用；
5. 应用：对所获数据进行加工，为决策和控制提供依据。

## 13. 4+1 视图模型

从5个不同视角来描述软件架构，每个视图只关心系统的一个侧面。

- 逻辑视图，主要支持系统的功能需求，即系统提供给最终用户的服务，在逻辑视图中，系统分解成一系列功能抽象，这些抽象主要来自问题领域。在OO技术中，通过抽象、封装和集成，可以用对象模型来代表逻辑视图，用类图来描述逻辑视图。
- 开发视图：UML 中的实现视图，软件模块的组织和管理。开发视图要考虑软件内部的需求。
- 进程视图：运行特性，主要关注**非功能需求**，性能和可用性。进程视图强调并发性、分布性、系统集成性和容错能力，以及逻辑视图中的 功能抽象如何适应进程结构等，它也定义了逻辑视图中的各个类的操作具体是在哪一个线程中被执行。进程视图可以描述成多层抽象，每个级别分别关注不同方面。
- 物理视图：UML 部署视图，拓扑结构、系统安装、通信问题
- **场景视图**：UML 用例视图，重要系统活动的抽象，使4个视图有机联系起来

## 16. 决策支持系统 DSS

决策支持系统包含的典型组成部件及对应的基本功能。

- 接口部分，即输入/输出的界面，是人机交互的窗口
- 模型管理子系统，具有存储、动态建模的功能。目前模型管理系统是通过模型库系统来实现的
- 知识管理子系统，集中管理决策问题领域的知识（规则和事实），包括知识的获取、表达和管理等功能

- 用户，用户可看作系统等一部分。DSS的用户主要是企业各层次的管理者和商业分析人员。

## 18. 缓存-数据库一致性问题

缓存和数据库的数据一致性问题，2种实现方案：

- 同步方案，更新数据时在同一事物内依次完成删除缓存、更新数据库、再写入缓存
- 异步方案，更新数据时在同一事物内首先通过消息队列发布待更新数据的消息给缓存更新服务，再更新数据库；缓存更新服务订阅消息队列，待收到更新事件执行缓存更新

数据量大，性能要求高时，推荐采用异步方案。

缓存分片的常用算法：

- 哈希算法，又称为散列算法，计算得到散列值（哈希值），按该值将数据分配到集群响应节点
- 一致性哈希算法，将哈希空间抽象为环形，计算哈希值，然后根据哈希值的位置沿圆环顺时针查找，将数据分配到第一个遇到的集群节点

一致性哈希算法有核心优点，适应服务器数量动态变化。

可扩展性，一致性哈希算法保证了增加或减少服务器时，数据存储的改变最少，相比传统哈希算法大大节省了数据移动的开销。

## 19. 布隆过滤器

布隆过滤器的原理是当一个元素被加入集合时，通过 K 个散列函数将这个元素映射成一个位数组中的 K 个点，把它置为1。检索时，只要看这些点是否为1，就大概知道集合中是否有它；如果这些点有任何一个0，则被检元素一定不在；如果都是1，则被检元素大很可能在。

优点：

- 增加和查询元素的时间复杂度为：O（K），与数据量大小无关
- 哈希函数之间没有关系，方便硬件并行计算
- 不需要存储元素本身，在某些对保密要求比较严格的场合有很大优势
- 在能够承受一定的误判时，布隆过滤器相比其他数据结构有很大的空间优势

缺点：

- 有误判性
- 不能获取元素本身
- 一般情况下不能从布隆过滤器中删除元素

## 20. MQTT 协议

MQTT 协议是一个物联网传输协议，它被设计用于轻量级的发布/订阅式消息传输，旨在为**低带宽**和**不稳定的网络**环境中的物联网设备提供可靠的网络服务。MQTT 协议针对低带宽网络、低计算能力设备做了特殊的优化，使得其能适应各类物联网场景。
